# 第0步：安装插件（只做一次，5分钟）

系统需要两个插件才能正常工作，首次使用前必须安装。

1. 按 `Win + S`，搜索 **PowerShell**，点击打开
2. 把下面这行命令**完整复制**，粘贴到 PowerShell 窗口，按回车：

```
New-Item -ItemType Directory -Force "$env:USERPROFILE\.config\opencode" | Out-Null; Set-Content "$env:USERPROFILE\.config\opencode\opencode.json" '{"plugin":["superpowers@git+https://github.com/obra/superpowers.git","oh-my-openagent"]}'
```

3. 看到光标回到下一行（没有报错）→ 成功
4. **重启 opencode**（关掉再重新打开），让插件生效
5. 看到 opencode 对话界面后继续第一步

  python -u tools/case_builder.py --all --embed-batch 128 2>&1 | Tee-Object logs\case_log.txt
  预计耗时（5000本）：约 3-4 小时

  预计耗时（5000本）：约 20-40 小时；建议先用 --limit 1000 处理1000本，后续再增量补充

# 第五步：提炼你的小说库（随时可做，强烈推荐）

打开 opencode，把下面这段话复制进去：

```

我的数据盘是：[填写你的数据盘符，比如 E]

请按以下顺序帮我完成：

第一步：安装格式支持库
pip install ebooklib mobi pdfminer.six python-docx

第二步：把路径写入 config.json（用 Python 脚本修改，不要手动编辑）

第三步：检测 GPU 状态，决定用哪种推理方式
执行：python -c "import torch; print('CUDA可用' if torch.cuda.is_available() else '仅CPU')"

- 如果显示"CUDA可用" → 直接进第四步
- 如果显示"仅CPU"，先检查我有没有 NVIDIA 独立显卡：
  执行：python -c "import torch; print(torch.version.cuda)"

    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121 -i https://pypi.tuna.tsinghua.edu.cn/simple
    安装完后重新验证：python -c "import torch; print('CUDA可用' if torch.cuda.is_available() else '仍然仅CPU')"

第四步：先用10本做测试，确认环境正常
python -u tools/batch_extract.py --skip-case --limit 10
看到 [DONE] 字样 → 正常，继续第五步

第五步：在 PowerShell 里启动两路提炼（必须用 PowerShell，不能用 CMD 黑色窗口）

路径一——11维度技法库（这个窗口跑）：

路径二——场景案例库（另开一个 PowerShell 窗口跑）：
根据第三步的结果选择命令：

台式独显（CUDA可用）：
python -u tools/case_builder.py --all --embed-batch 128 2>&1 | Tee-Object logs\case_log.txt
预计耗时（5000本）：路径一 3-6小时 + 路径二 3-4小时，合计约 6-10小时

注意散热：放在硬质平面上，不要遮住底部进风口

```

# 遇到问题怎么办

**首先**：把你看到的内容或报错，直接粘贴给 opencode 或 Claude Code，描述你在做什么，它会帮你解决。不需要自己看懂报错。

---

**Q：Docker Desktop 显示 "WSL needs updating"**

一条命令解决（在 PowerShell 管理员模式下）：

```
wsl --update
```

跑完后回 Docker Desktop 点 **Try Again**。

---

**Q：Qdrant 容器没有启动**

在 Claude Code 里说：

```
帮我检查 Qdrant 容器状态并重新启动
```

---

**Q：AI 说"找不到世界观配置"**

```
帮我检查当前小说"[你的小说名]"的世界观配置是否正常
```

如果丢失，重新描述世界观，通过对话重建即可，不影响已有章节。

---

**Q：BGE-M3 模型下载太慢或失败**

在 opencode 里说：

```
BGE-M3 模型下载失败，帮我用备用镜像重试
```

或找老师要离线模型文件，解压后在 config.json 里配置 `model_path`。

---

**Q：AI 生成内容感觉很普通**

初期正常，系统还不了解你的品味：

1. 直接说："这个太普通了，换个思路"
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后会明显改善

---

**Q：我想写另一本小说**

```
使用 novel-workflow，切换到我的另一本小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。

---

**Q：提炼过程中电脑突然自动重启**

笔记本 GPU 长时间满载会过热，属正常保护机制。重启后重新执行同一命令，已处理的书自动跳过。

解决方案：在 opencode 里说：

```
帮我用更小的 embed-batch 参数重新运行 case_builder，我的电脑之前过热重启了
```

---

**Q：提炼命令里 GPU 不工作（显示"使用 CPU 推理"）**

必须在 **PowerShell**（有 `PS C:\>` 前缀的窗口）里运行，不能用 CMD 黑色窗口。

# 附录：config.json 配置说明

打开 `config.json`（记事本即可），找到并修改以下内容。**其他内容不要动。**

> 通常不需要手动修改这个文件，安装时 opencode 会自动帮你改好。只有在出问题或需要调整时才看这里。

---

### ① current_world（必须改）

```json
"worldview": {
    "current_world": "天道余烬"
```

改成**你的小说名**（和你告诉系统的名字完全一致）。

---

### ② realm_order（必须改）

```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"]
```

改成**你小说的境界体系**，从低到高排列。如果你的小说不需要境界检测，留空数组：`"realm_order": []`

---

### ③ model_path（只有使用离线模型时才改）

```json
"model": {
    "model_path": null
```

如果老师给了离线模型文件：

```json
"model_path": "D:/models/bge-m3"
```

注意路径用正斜杠 `/`。

---

### ④ 路径配置

```json
"paths": {
    "skills_base_path": "C:/Users/你的用户名/.agents/skills",
    "cache_dir":        "X:/众生界_cache",
    "temp_dir":         "X:/众生界_temp",
    "mobi_temp_dir":    "X:/tmp_mobi"
},
"novel_sources": {
    "directories": ["X:\\小说资源"]
},
"extractor": {
    "output_dir": "X:/novel_extracted"
}
```

把所有 `X:/` 改成你的数据盘符（比如 `E:/`）。`skills_base_path` 里的 `你的用户名` 换成你的 Windows 用户名（PowerShell 输入 `$env:USERNAME` 查看）。

**磁盘空间要求：**

| 目录 | 用途 | 建议预留 |
|------|------|---------|
| `mobi_temp_dir` | mobi 格式解压临时文件 | 5 GB |
| `extractor.output_dir` | 11维度提炼输出 | 80 GB |
| Qdrant 存储 | 向量数据库 | 50 GB |
| `cache_dir` | 系统缓存 | 10 GB |

> `mobi_temp_dir` 绝对不能放 C 盘，批量处理时临时文件可能超过 50 GB。

---

### 其他常用配置（通常不需要改）

| 配置项 | 默认值 | 什么时候改 |
|--------|--------|-----------|
| `qdrant_host` | `"localhost"` | 换成云端 Qdrant 时 |
| `qdrant_port` | `6333` | 改了 Docker 端口映射时 |
| `auto_sync` | `true` | 保持 true |
| `skip_rules` | `[]` | 某条校验规则总是误报时加进去 |
