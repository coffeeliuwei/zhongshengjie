## 2. 第一步：安装基础软件

### 2.1 安装 Python 3.12

**第一步：下载**

打开浏览器，访问：`https://www.python.org/downloads/`

找到 **Python 3.12.x**（x 是具体版本号，选最新的即可），点击下载 **Windows installer (64-bit)**。

**第二步：安装**

1. 双击下载的 `.exe` 文件
2. **重要：** 在安装界面底部，勾选 **"Add python.exe to PATH"**（这个必须勾！）
3. 点击 **"Install Now"**
4. 等待安装完成，点击 **"Close"**

**第三步：验证安装**

按 `Win + R`，输入 `cmd`，回车，打开黑色窗口（命令提示符）。

输入以下内容并回车：
```
python --version
```

如果看到 `Python 3.12.x`，说明安装成功。

---

### 2.2 安装 Git

**第一步：下载**

访问：`https://git-scm.com/download/win`

下载 **64-bit Git for Windows Setup**。

**第二步：安装**

双击安装文件，全程点 **"Next"**（默认设置即可），最后点 **"Finish"**。

**第三步：验证**

在命令提示符中输入：
```
git --version
```

看到 `git version x.x.x` 即为成功。

## 3. 第二步：获取项目文件

### 3.1 选择一个存放位置

建议把项目放在一个路径简单的位置，例如 `D:\novels\` 或 `C:\Users\你的用户名\novels\`。

**避免放在：** 含有中文的路径、桌面、Downloads 文件夹。

### 3.2 下载项目

在命令提示符中，先切换到你想存放的目录：

```
d:
mkdir novels
cd novels
```

然后下载项目（地址从老师处获取）：

```
git clone https://github.com/[老师提供的地址]
```

> 💡 如果速度很慢，可以询问老师是否有其他下载方式。

下载完成后，进入项目目录：

```
cd 众生界
```

你应该能看到 `core/`、`modules/`、`tools/` 等文件夹，说明下载成功。

### 3.3 复制配置文件

项目里有一个 `config.example.json`（模板配置），需要复制成 `config.json`（正式配置）：

```
copy config.example.json config.json
```

## 4. 第三步：安装项目依赖

### 4.1 理解依赖是什么

"依赖"就是这个系统需要用到的其他工具包，类似于手机 APP 需要运行时库一样。

### 4.2 安装（关键步骤，耐心等待）

在命令提示符中，确保你在项目目录内（路径显示 `D:\novels\众生界>`），然后执行：

```
pip install -r requirements.txt
```

这个过程会下载很多包，大约需要 **5-15 分钟**，看网速。如果提示 **successfully installed** 就是成功了。

> ⚠️ **如果速度很慢：** 可以换成国内镜像源：
> ```
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 4.3 下载 AI 语言模型（BGE-M3）

系统使用 BGE-M3 模型来理解文字。首次运行时会自动下载（约 2-3 GB），但**在中国大陆可能需要特殊网络**。

如果老师提供了模型文件，直接告诉你放在哪里，按指引操作即可。

验证是否能正常下载（在项目目录执行）：
```
python -c "from sentence_transformers import SentenceTransformer; print('模型库OK')"
```

看到 `模型库OK` 即可（真正下载在首次使用时发生）。

## 5. 第四步：安装向量数据库 Qdrant

Qdrant 是一个存储"小说知识"的数据库，你所有的世界观、写作技法、案例都存在这里。

### 5.1 下载 Qdrant

找到最新版本，下载 `qdrant-x86_64-pc-windows-msvc.zip`（Windows 版本）。

### 5.2 解压并放置

将压缩包解压，得到 `qdrant.exe`。建议放在：
```
D:\tools\qdrant\qdrant.exe
```

打开一个**新的**命令提示符窗口（不要关闭），进入 qdrant 所在目录：

```
d:
cd tools\qdrant
qdrant.exe
```

看到类似以下内容说明启动成功：

## 8. 第七步：初始化你的小说项目

到这一步，系统基本安装完毕。现在来创建你自己的小说。

### 8.1 进入项目目录

在命令提示符中，进入项目：
```
d:
cd novels\众生界
```

### 8.2 运行初始化向导

执行以下命令（把"我的玄幻世界"替换成你的小说名）：
```
python tools/init_novel.py --name "我的玄幻世界"
```

系统会自动：
- 在 `config/worlds/` 创建 `我的玄幻世界.json`（你的世界观配置文件）
- 更新 `config.json`，设置当前小说为"我的玄幻世界"

你会看到类似：
```
=== 初始化小说项目：我的玄幻世界 ===

[创建] 世界配置: config/worlds/我的玄幻世界.json
[更新] config.json worldview.current_world → '我的玄幻世界'

=== 初始化完成 ===
下一步：
1. 编辑世界观配置：config/worlds/我的玄幻世界.json
```

### 8.3 填写你的世界观

用记事本（或 VS Code）打开 `config/worlds/我的玄幻世界.json`。

文件结构如下，按注释修改：

```json
{
  "world_name": "我的玄幻世界",
  "world_type": "xianxia",
  "description": "在这里描述你的世界观，2-3句话即可",
  "power_systems": {
    "主要力量体系名": {
      "description": "简单描述这个力量体系",
      "levels": ["入门级", "进阶级", "高阶级", "巅峰级"]
    }
  },
  "factions": {
    "主要势力名": {
      "description": "势力简介",
      "alignment": "正义/邪恶/中立"
    }
  },
  "key_characters": {
    "主角名": {
      "role": "主角",
      "personality": "性格描述",
      "background": "背景故事"
    }
  },
  "core_principles": {
    "theme": "你的小说核心主题，如：成长、救赎、权谋",
    "tone": "叙事基调，如：热血、悲壮、轻松"
  }
}
```

> 💡 **不需要一次填完**，先填你已经想好的部分，其余可以随时补充。

## 10. 各模块使用指南

### 10.1 主模块：章节创作工作流

**触发方式：**
```
使用 novel-workflow，写第一章
```
或
```
使用 novel-workflow，继续写第三章，这章重点是主角觉醒
```

**工作流程（系统自动完成）：**
```
阶段0：需求澄清（和你讨论这章的重点）
    ↓
阶段1-3：世界观/人物/剧情素材检索
    ↓
阶段4：灵感发散（AI 提出创意方向）
    ↓
阶段5：鉴赏师审核创意
    ↓
阶段5.5：三方协商（你参与决策保留哪些创意）
    ↓
阶段6：评估师检查质量
    ↓
阶段7（如需）：你可以推翻评估结果，坚持自己的判断
    ↓
阶段8：经验写入（系统学习这次创作的经验）
    ↓
输出：完整章节文件
```

**三方协商阶段（你需要参与）：**

系统会展示 AI 提出的创意建议，你来决定：
- 哪些必须保留（输入 Y 或序号）
- 哪些不要（输入 N 或直接跳过）
- 这一章跳过协商（直接回车）

**示例对话：**
```
你：使用 novel-workflow，写第一章，主角初入江湖

AI：好的，我们来讨论第一章的重点……
    你希望这章侧重：
    A. 主角的内心成长
    B. 精彩的战斗出场
    C. 悬念铺垫，吊足胃口

你：A，同时希望有一个印象深刻的人物出场

AI：明白，我有几个创意想法……
    【鉴赏师建议1】…
    【鉴赏师建议2】…
    你想保留哪些？

你：保留第1条，第2条改一改

AI：好的……（继续流程，最终输出完整章节）
```

## 11. 构建你的小说体系

### 11.1 建立世界观的正确顺序

**推荐顺序：**

```
第一周：确定世界观核心
├── 填写 config/worlds/你的小说.json 基础内容
├── 用 novel-workflow 写 1-2 章试水
└── 根据创作中的发现补充设定

第二周：建立知识库
├── 把你喜欢的写作片段通过 inspiration-ingest 录入
├── 用检索模块确认知识库有内容了
└── 继续创作，系统会自动调用

第三周以后：形成正循环
├── 每次创作 → 系统学习经验 → 下次更精准
├── 你推翻评估 → 系统记录你的品味 → 越来越懂你
└── 积累越多，辅助越强
```

### 11.2 你的品味如何塑造系统

系统有一个"记忆点库"，记录你每次创作中**喜欢**和**不喜欢**的表达。

**让系统了解你的方法：**

当 AI 提出你不喜欢的创意时，明确说出：
```
这个创意不好，太俗套了，我不喜欢这种套路
```

当某段写得很好时：
```
这段写得很棒，这个风格我喜欢，记住它
```

随着时间积累，系统对你品味的理解越来越准确，提出的创意也越来越符合你的风格。

### 11.3 世界观不断深化

你的 `config/worlds/你的小说.json` 应该是一个**活文件**，随着你的创作不断补充：

- 写到新势力 → 加进 `factions`
- 想到新角色 → 加进 `key_characters`
- 修改了设定 → 直接改对应字段

每次创作前，写手会自动加载最新的世界观配置。

### 11.4 章节经验的积累

每完成一章，系统会在 `章节经验日志/` 目录生成一个 JSON 文件，记录：
- 这章用了哪些技法
- 哪些写法有效
- 哪些写法不好
- 给下一章的建议

你可以直接打开这些文件读一读，了解自己的创作规律。

## 12. 常见问题

### Q1：运行命令出现乱码或中文显示问题

**解决：** 在命令提示符中先执行：
```
chcp 65001
```
然后再运行其他命令。

---

### Q2：pip install 一直报错

**常见原因和解决方法：**

- **网络问题：** 换镜像源
  ```
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- **Python 版本不对：** 确认 `python --version` 显示 3.12.x
- **权限问题：** 用管理员身份运行命令提示符

---

### Q3：启动 Claude Code 报错

**检查清单：**
1. API Key 是否正确配置：`claude config list`
2. 网络是否能访问 Anthropic 服务
3. 询问老师是否有网络代理设置

---

### Q4：Qdrant 启动后立刻关闭

**解决：**
1. 确认是在 qdrant.exe 所在目录运行
2. 检查 6333 端口是否被占用：`netstat -an | findstr 6333`
3. 如有占用，关闭占用进程后重试

---

### Q5：写手说"找不到世界观配置"

**解决：**
1. 确认 `config.json` 中 `worldview.current_world` 已设置为你的小说名
2. 确认 `config/worlds/你的小说名.json` 文件存在
3. 重新运行：`python tools/init_novel.py --name "你的小说名"`

---

### Q6：AI 生成的内容感觉很普通

**这很正常，系统需要"训练"你的品味：**
1. 明确告诉系统你不喜欢什么
2. 多用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后，系统对你品味的理解会明显提升

---

### Q7：我想换一个小说来写

执行以下命令（换成你的新小说名）：
```
python tools/init_novel.py --name "新小说名称"
```

如果这个小说之前没有建过，会自动创建配置文件。  
如果想切换回已有的小说，直接运行同样命令即可（不会覆盖已有的世界观配置）。
