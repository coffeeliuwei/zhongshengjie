```
阶段 1：检查基础环境

依次检查以下工具是否已安装：

  python --version   （期望：Python 3.12.x）
  git --version      （期望：任意版本号）
  docker --version   （期望：任意版本号）
  docker ps          （期望：命令能执行，说明 Docker 守护进程在运行）

判断逻辑：
- python 版本不是 3.12.x → 告诉我去 https://www.python.org/downloads/ 下载 3.12，安装时勾选"Add python.exe to PATH"，装完让我重新打开 opencode 再粘贴这段提示词
- git 未安装 → 告诉我去 https://git-scm.com/download/win 下载安装，一路 Next
- docker 未安装或 docker ps 报错 → 告诉我去 https://www.docker.com/products/docker-desktop/ 安装 Docker Desktop，安装后重启电脑，重启后右下角出现鲸鱼图标再重新开始

等我回答后，把我说的盘符记为 [数据盘]（比如我说 E，就记为 E），后续所有命令里的 [数据盘] 全部替换成实际盘符，然后继续阶段 2。
```

## 写章节

在 Claude Code 里输入：


**创作过程中你会参与的关键环节：**

# 第三部分：让系统越来越懂你

> 这部分**不需要一次性完成**，边写边补效果最好。

## 你需要逐步建立的4类内容

| 类型 | 存在哪里 | 作用 |
|------|---------|------|
| 大纲 | `总大纲.md` + `章节大纲/` | 写手创作时的故事地图 |
| 设定 | `config/worlds/你的小说.json` | 世界规则、人物、势力 |
| 技法 | `创作技法/`（11个维度分类） | 写手参考的写法库 |
| 评估标准 | 品味记忆（通过对话建立） | 让评估师符合你的风格 |

**核心原则：你只需要说话，系统自动判断是哪类内容并存入对应位置。**

# 常见问题

**Q：运行命令出现乱码**

```
chcp 65001
```

执行这行后再运行其他命令。

---

**Q：pip install 一直失败**

- 确认 Python 版本是 3.12.x（`python --version`）
- 加国内镜像：命令里已有 `-i https://pypi.tuna.tsinghua.edu.cn/simple`
- 权限问题：右键命令提示符，选"以管理员身份运行"

---

**Q：Docker Desktop 打不开**

1. 确认 Windows 版本（开始菜单搜"winver"），需要是 2004 或更高
2. BIOS 中需要开启虚拟化（一般默认开启）
3. 尝试重装 Docker Desktop

---

**Q：Qdrant 容器没有启动**

```
docker ps -a
```

- 看到 qdrant 状态是 `Exited` → `docker start qdrant`
- 完全没有 qdrant 容器 → 告诉 opencode "Qdrant 容器不见了"，让它重新执行启动命令
- 看报错原因 → `docker logs qdrant`

---

**Q：AI 说"找不到世界观配置"**

```
帮我检查当前小说"天道余烬"的世界观配置是否正常
```

如果丢失，重新描述世界观，通过对话重建即可，不影响已有章节。

---

**Q：BGE-M3 模型下载太慢或失败**

1. 确认设置了镜像：`$env:HF_ENDPOINT = "https://hf-mirror.com"`
2. 实在不行，找老师要离线模型文件，按附录A配置

---

**Q：AI 生成内容感觉很普通**

初期正常，系统还不了解你的品味：

1. 明确告诉 AI 不喜欢什么："这个太普通了，换个思路"
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后明显改善

---

**Q：我想写另一本小说**

```
使用 novel-workflow，切换到我的另一本小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。

---

**Q：安装中途卡住不知道怎么办**

直接在 opencode 对话框里描述你看到的内容或报错信息，它会帮你分析解决。不需要自己看懂报错。

# 附录A：config.json 配置说明

打开 `config.json`（记事本即可），找到并修改以下内容。其他内容**不要动**。

---

### ① current_world（必须改）

```json
"worldview": {
    "current_world": "天道余烬"
```

改成**你的小说名**（和你告诉系统的名字完全一致）。

---

### ② realm_order（必须改）

```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"]
```

改成**你小说的境界体系**，从低到高排列。系统用这个来检测"境界是否合法提升"。

如果你的小说不需要境界检测，留空数组即可：`"realm_order": []`

---

### ③ model_path（只有离线时才改）

```json
"model": {
    "model_path": null
```

如果老师给了离线模型文件，改成模型文件夹的路径：

```json
"model_path": "D:/models/bge-m3"
```

注意路径用正斜杠 `/` 而不是反斜杠 `\`。

---

---

### ④ 路径配置（必须改）

把以下配置里的 `X` 全部换成你的**数据盘符**（即安装时告诉 opencode 的那个盘，不能用 C 盘）：

```json
"paths": {
    "skills_base_path": "C:/Users/你的用户名/.agents/skills",
    "cache_dir":        "X:/众生界_cache",
    "temp_dir":         "X:/众生界_temp",
    "mobi_temp_dir":    "X:/tmp_mobi"
},
"novel_sources": {
    "directories": ["X:\\小说资源"]
},
"extractor": {
    "output_dir": "X:/novel_extracted"
}
```

**修改方法：**
- 把所有 `X:/` 改成你的数据盘符（比如 `D:/` 或 `E:/`）
- `skills_base_path` → 把 `你的用户名` 改成你的 Windows 用户名（PowerShell 输入 `$env:USERNAME` 查看）
- `novel_sources.directories` → 改成你存放小说文件的实际目录

**磁盘空间要求：**

| 目录 | 用途 | 建议预留空间 |
|------|------|------------|
| `mobi_temp_dir` | mobi 格式小说解压临时文件 | 5 GB |
| `extractor.output_dir` | 11维度提炼输出 | 80 GB |
| Qdrant（Docker volume）| 向量数据库 | 50 GB |
| `cache_dir` | 系统缓存 | 10 GB |

> ⚠️ `mobi_temp_dir` 绝对不能放 C 盘：mobi 解压时每本书最大 500MB，批量跑几百本会把系统盘塞满导致电脑崩溃。

---

### 其他常用配置（通常不需要改）

| 配置项 | 默认值 | 什么时候改 |
|--------|--------|-----------|
| `qdrant_host` | `"localhost"` | 换成云端 Qdrant 时改 |
| `qdrant_port` | `6333` | 改了 Docker 端口映射时才改 |
| `auto_sync` | `true` | 保持 true，设定更新后自动同步 |
| `skip_rules` | `[]` | 某条校验规则总是误报时，把规则编号加进去 |

# 附录B：手动安装步骤（备用）

## 步骤 1：安装 Python

**去哪里下载：** `https://www.python.org/downloads/`

选 **Python 3.12.x**，点击下载 Windows installer (64-bit)。

**安装时最重要的一步：**

> ⚠️ 打开安装程序后，在最底部看到 **"Add python.exe to PATH"** 选项，**必须打勾！**
> 然后点 "Install Now"。

**验证是否成功：**

按 `Win + R`，输入 `cmd`，回车。在黑色窗口里输入：

```
python --version
```

看到 `Python 3.12.x` → 成功。

---

## 步骤 2：安装 Git

**去哪里下载：** `https://git-scm.com/download/win`

下载 64-bit 版本，双击安装，一路点 **Next**，最后点 Finish。

**验证：**

```
git --version
```

看到版本号 → 成功。

---

## 步骤 3：安装 Docker Desktop

Docker 用来运行数据库，是必须安装的。

**去哪里下载：** `https://www.docker.com/products/docker-desktop/`

点 **Download for Windows**，下载完双击安装。

> 安装过程中可能提示安装 WSL2（Windows 的 Linux 子系统），点"安装"或"继续"。
> 安装完成后**必须重启电脑**。

重启后桌面右下角会出现鲸鱼图标 🐳，说明 Docker 在运行。

**验证：**

```
docker --version
```

看到版本号 → 成功。

---

## 步骤 4：下载项目文件

打开 PowerShell，输入以下命令（一行一行执行）：

```
mkdir D:\novels
cd D:\novels
git clone https://github.com/[老师提供的地址]
cd 众生界
```

看到 `core/`、`modules/`、`tools/` 等文件夹 → 成功。

## 步骤 5：安装依赖和 AI 模型

**5.1 安装依赖包**（在项目目录内执行）

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

大约 5-15 分钟，看到 `Successfully installed` → 成功。

```
$env:HF_ENDPOINT = "https://hf-mirror.com"
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-m3'); print('模型下载完成')"
```

看到 **模型下载完成** → 成功（需要 10-30 分钟）。

> 如果老师提供了离线模型文件，见附录A的离线配置说明。

## 步骤 6：启动 Qdrant 数据库

Qdrant 是用来存储所有小说知识的数据库。

**6.0 配置镜像加速**（国内网络必做，否则下一步会超时）

打开 Docker Desktop → 右上角齿轮图标 → **Docker Engine** → 在 JSON 里加入以下内容：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```

点 **Apply & Restart**，等 Docker 重启完成（右下角鲸鱼图标变绿）。

> 如果老师提供了学校专用镜像地址，优先填学校的地址，效果更快。

**6.1 拉取 Qdrant 镜像**（只做一次）

```
docker pull qdrant/qdrant
```

> 如果拉取过程中卡住超过 3 分钟没有进度，按 `Ctrl+C` 停止，确认步骤 6.0 的镜像加速已生效后重试。

**6.2 创建数据存放目录**（只做一次，放大容量数据盘，需要约 35-50 GB 空间，不要放 C 盘）

把下面命令里的 `X` 换成你的数据盘符（比如 D、E、F）：

```
mkdir X:\qdrant_storage
```

**6.3 启动 Qdrant 容器**（只做一次，之后自动启动）

同样把 `X` 换成你的数据盘符：

```
docker run -d --name qdrant --restart unless-stopped -p 6333:6333 -p 6334:6334 -v X:\qdrant_storage:/qdrant/storage qdrant/qdrant
```

**验证：**

```
docker ps
```

看到 qdrant 容器状态是 `Up` → 成功。

也可以打开浏览器访问 `http://localhost:6333/dashboard`，看到界面 → 成功。

## 步骤 11：提炼外部小说库（建议完成）

把你有的小说资源（TXT / EPUB / MOBI / PDF / DOCX 格式）放到一个目录，比如 `X:\小说资源\`（X 换成你的数据盘符），然后执行：

**安装格式支持库：**

```
pip install ebooklib mobi pdfminer.six python-docx
```

**把路径写入配置**（把下面的 `X` 换成你的数据盘符）：

```
python -c "
import json
with open('config.json', 'r', encoding='utf-8') as f:
    cfg = json.load(f)
cfg['novel_sources']['directories'] = ['X:\\\\小说资源']
cfg['extractor']['output_dir'] = 'X:/novel_extracted'
with open('config.json', 'w', encoding='utf-8') as f:
    json.dump(cfg, f, ensure_ascii=False, indent=2)
print('配置已更新')
"
```

**先用测试模式验证（每个维度只跑50本）：**

```
python -u tools/batch_extract.py --skip-case --limit 50
```

看到 `[DONE] 批量提炼完成` → 正常，然后跑全量（X 换成你的数据盘符）：

```
python -u tools/batch_extract.py --skip-case 2>&1 | Tee-Object X:\extract_log.txt
```

> 💡 **可以中断后继续**：重新执行同一命令，已处理的书自动跳过。
> 💡 **建议睡前开始**：5000 本约需 3-6 小时，磁盘占用约 55 GB（数据盘需预留 80 GB）。
> 💡 案例库是写作质量的核心燃料，建议在开始创作前完成这步。
