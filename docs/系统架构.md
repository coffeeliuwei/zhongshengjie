## 二、信息储备路径（novel-inspiration-ingest）

```

        │
        ▼

        │
        ▼ Q2 资料层次路由
        │
   ┌────┴──────┬────────────┬───────────┬───────────┐
   ▼           ▼            ▼           ▼           ▼
 设定        总大纲       章节大纲    技法+案例    术语/弃用
   │           │            │           │
   ▼           ▼            ▼           ├──→ 创作技法/99-从小说提取/{维度}.md

sync-novel  sync-total  sync-chapter  sync-technique + sync-case

## 四、批量外部小说提炼管道

```
python tools/batch_extract.py
              │
              ▼
Step 1（案例提炼）
case_builder.py --all --embed-batch 128     ← 一键：convert → extract → sync
（等价于分步：--convert → --extract → --sync，学生只需此一条命令）
关键词匹配+28场景
C4清洗+Bigram熵
MinHash LSH去重
[Step 2 已移除：unified_case_extractor.py 在 P1 清理中删除]
              │
              ▼
        E:/case-library/cases/
        {场景类型}/{novel_slug}.json
              │
   ┌──────────┴──────────────────────────────┐
   ▼                                         ▼
Step 3                                    Step 4
unified_extractor.py --force          scene_discoverer.py
11维度并行提取：                        从未分类片段发现新场景类型
  technique → writing_techniques_v2    → 更新 SCENE_TYPES
  character_relation → character_relation_v1
  dialogue_style → dialogue_style_v1
  emotion_arc → emotion_arc_v1
  power_vocabulary → power_vocabulary_v1
  foreshadow_pair → foreshadow_pair_v1
  worldview_element → worldview_element_v1
  power_cost → power_cost_v1
  author_style → author_style_v1
              │
              ▼
Step 5: case_builder.py --sync → case_library_v2
Step 6: sync_extracted_to_qdrant.py → 8个维度 v1 collection

数据质量管道（case_builder 内嵌，详见 §十一）：
  原始文本
    → _clean_lines()              C4 行级过滤（广告/版权行）
    → 长度门槛                    100-5000字
    → Q1 TF-IDF关键词加权评分     高频虚词降权，精确词升权
    → Q2 题材检测                 三段采样，10种题材+未分类
    → Q3 Embedding Delta边界验证  余弦距离<0.12丢弃（Schneider 2021）
    → Q4 Zero-shot语义校验        28种锚向量，max_sim<0.20丢弃（ACM 2022）
    → _bigram_entropy()           熵<5.0扣分，>8.0加分
    → _filter_near_duplicates()   MinHash LSH threshold=0.80

格式支持（两条路径均已覆盖）：
  .txt  → 多编码尝试 (utf-8/gbk/gb2312/utf-16)
  .epub → ebooklib
  .mobi → mobi库 → epub → ebooklib
  .pdf  → pdfminer.six
  .docx → python-docx

> ⚠️ 2026-04-28：路径一（case_builder）新增 `_read_novel()` 统一分发方法，格式支持与编码方案已完全对齐路径二（base_extractor）。
> 路径一 `extract_cases` 同步新增增量标记：`E:/case-library/converted/.extract_progress/.processed_{novel_id}`，支持断点续提；首次转换不再因程序崩溃丢失已处理进度。
```

## 十、sync_extracted_to_qdrant 批量入库优化（2026-04-27）

`python tools/sync_extracted_to_qdrant.py --skip-existing --embed-batch 128`

### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--embed-batch` | 20（配置值）| embedding 推理 batch size；有 GPU 建议 128 |
| `--upsert-batch` | 500 | 每次写入 Qdrant 的条数 |
| `--max-length` | 512 | tokenizer 序列长度；短文本维度可设 128 加速 |

### 三项性能优化

| 优化 | 实现 | 效果 |

## 十一、内容提炼质量优化策略（两条路径）

### 11.1 路径一：case_builder.py 场景案例提炼

#### 分值体系

```
原始片段
  │
  ├─ Q0 基础门槛（先于分值，不通过直接丢弃）
  │     长度 100–5000字（config: content_max_len）
  │     C4 行级清洗（广告/版权/无效行）
  │     MinHash LSH 近重复过滤（config: minhash_threshold=0.80）
  │
  └─ Q1-Q4 分值评估（base=4.5，threshold=5.0）
        │
        ├─ Q1  TF-IDF 关键词加权
        │      · 高频虚词降权：却×0.05、竟×0.1（消除心理场景假阳性）
        │      · 精确转折词升权：反转×1.5
        │      · 关键词命中数量 → score 加减
        │
        ├─ Q2  题材检测（三段采样）
        │      · 头/中/尾各取5000字（config: genre_sample_size）
        │      · 命中次数 < 3（config: genre_detection_threshold）→ "未分类"

        │      · 滑动窗口3段（config: embedding_window_size）
        │      · 相邻窗口 BGE-M3 余弦距离 < 0.12（config: boundary_delta_threshold）
        │        → 判定为场景中段噪音，丢弃
        │      · 依据：Schneider et al. 2021 (CEUR-WS Vol-3001)
        │
        └─ Q4  Zero-shot 语义场景类型校验
               · 28种场景类型描述预计算为锚向量（BGE-M3，1024维）

               · embedding 判定与关键词判定不符 → 覆盖 scene_type

#### sync_to_vectorstore 性能优化（2026-04-28 对齐路径二）

`python tools/case_builder.py --sync [--batch-size 128] [--embed-batch 128] [--skip-existing]`

| 优化 | 实现 | 说明 |
|------|------|------|
| embed batch 来自 config | `get_batch_size()` → 默认 128 | CLI `--embed-batch` > config > 128，不再硬编码 |
| HF_HOME 重定向 | `os.environ["HF_HOME"]` = `config.model.hf_cache_dir` | 模型缓存到 E 盘，防止 C 盘爆满 |
| HNSW 上传期间禁用 | `indexing_threshold=0`，完成后恢复 20000 触发后台重建 | 消除"越写越慢"问题（同路径二 §十）|
| upsert 502 自动重试 | 最多 5 次，间隔递增 5s | Qdrant 内存抖动不崩全程 |
| embed + upsert 流水线 | `ThreadPoolExecutor(1)` 异步上传，GPU 并行计算下批 | Qdrant I/O 基本被隐藏 |
| --skip-existing | 检查 Qdrant 条数是否已一致，一致则跳过 | 断点续跑 |

> CLI 参数：`--batch-size`（Qdrant upsert 批次，默认 128），`--embed-batch`（embedding 推理批次，默认 128），`--skip-existing`（增量模式）。

#### 路径一全流程性能参考（5000本，RTX 3070 Ti）

| 阶段 | 优化前 | 优化后 | 加速手段 |
|------|--------|--------|---------|
| `--convert` | ~3 小时（单线程 I/O）| ~40-60 分钟 | 非mobi → `ThreadPoolExecutor(workers=8)`；mobi → `ProcessPoolExecutor(workers=4)` |
| `--extract` | ~8 小时（逐候选 encode）| ~2 小时 | Q3/Q4 批量推理：每本书 n 次 → 2 次 BGE-M3 调用 |
| `--sync` | ~1 小时（HNSW 实时建索引）| ~20 分钟 | HNSW 禁用 + upsert 流水线 |
| **合计** | **~12 小时** | **~3 小时** | `--all --embed-batch 128` 一键触发 |

> mobi 格式（本库占 54%）原因：`mobi.extract()` 内部调 `tempfile.mkdtemp()`，依赖全局 `tempfile.tempdir`，多线程竞争。改用 `ProcessPoolExecutor`，每个进程有独立内存，无竞争，4进程并行约 40 分钟处理 3237 本。模块级 `_mobi_to_txt(task)` worker 函数负责具体提取（必须 pickle 可序列化）。
