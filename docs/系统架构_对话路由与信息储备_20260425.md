## 二、信息储备路径（novel-inspiration-ingest）

### 2.1 内部路由流程

```
用户扔进任何内容
        │
        ▼
┌───────────────────────────────────────────────────────────────┐
│  阶段1  归档：写入素材库/raw/{slug}/source.md                   │
│  阶段2  六问分类（Q1 素材类型 / Q2 资料层次 / Q3-6 其他）       │
│  阶段3  载入世界观上下文（读 总大纲.md + 设定/*.md）             │
└───────────────────────────────────────────────────────────────┘
        │
        ▼ Q2 资料层次路由
        │
   ┌────┴──────┬────────────┬───────────┬───────────┐
   ▼           ▼            ▼           ▼           ▼
 设定        总大纲       章节大纲    技法+案例    术语/弃用
   │           │            │           │
   ▼           ▼            ▼           ├──→ 创作技法/99-从小说提取/{维度}.md
设定/*.md   总大纲.md   章节大纲/       └──→ .case-library/cases/99-从小说提取/
                        第N章-章名.md         {维度}/{slug}.md
   │           │            │           │
   ▼           ▼            ▼           ▼
sync-novel  sync-total  sync-chapter  sync-technique + sync-case
```

### 2.2 Sync 命令速查表

## 四、批量外部小说提炼管道

```
python tools/batch_extract.py
              │
   ┌──────────┴──────────┐
   ▼                     ▼
Step 1                Step 2
case_builder.py        unified_case_extractor.py
--extract              --extract
关键词匹配+28场景       语义分组(BGE-M3 Ex3)
C4清洗+Bigram熵         语义去重(threshold=0.6)
MinHash LSH去重
   │                     │
   └──────────┬──────────┘
              ▼
        .case-library/cases/
        {场景类型}/{novel_slug}.json
              │
   ┌──────────┴──────────────────────────────┐
   ▼                                         ▼
Step 3                                    Step 4
unified_extractor.py --force          scene_discoverer.py
11维度并行提取：                        从未分类片段发现新场景类型
  technique → writing_techniques_v2    → 更新 SCENE_TYPES
  character_relation → novel_settings_v2
  dialogue_style → dialogue_style_v1
  emotion_arc → emotion_arc_v1
  power_vocabulary → power_vocabulary_v1
  foreshadow_pair → foreshadow_pair_v1
  worldview_element → novel_settings_v2
  power_cost → power_cost_v1
  author_style → author_style_v1
              │
              ▼
Step 5: sync_to_qdrant.py --docker --no-resume → case_library_v2
Step 6: migrate_lite_resumable.py × 9个维度 → 各自 collection

数据质量管道（case_builder 内嵌）：
  原始文本
    → _clean_lines()      C4 行级过滤（广告/版权行）
    → 长度门槛            100-5000字
    → _bigram_entropy()   熵<5.0扣分，>8.0加分
    → _filter_near_duplicates()  MinHash LSH threshold=0.85

格式支持：
  .txt  → 多编码尝试 (utf-8/gbk/gb2312/utf-16)
  .epub → ebooklib
  .mobi → mobi库 → epub → ebooklib
  .pdf  → pdfminer.six
  .docx → python-docx
```
