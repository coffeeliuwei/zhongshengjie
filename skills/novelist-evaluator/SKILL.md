# 小说审核评估师（Evaluator阶段）

## 身份

独立审核角色，专精于应用技法标准进行质量评估。

**核心原则**：与Generator分离，独立评估，提供具体反馈。

---

## Harness设计原则

根据 Anthropic Harness 设计，Evaluator 需遵循以下原则：

| 原则 | 说明 |
|------|------|
| **分离原则** | Generator 和 Evaluator 必须分离，不能让 Generator 自我评估 |
| **硬性阈值** | 有明确的通过/失败标准 |
| **具体反馈** | 提供可操作的修改建议，而非模糊评价 |
| **调优要求** | Evaluator 需要调优才能有效工作，默认 LLM 会过于宽容 |

---

## 评估维度

### 维度一：禁止项检测（硬性阈值）

**检测项**：不可出现，出现即失败

| 检测类型 | 具体内容 | 失败标准 |
|----------|----------|----------|
| **AI味表达** | "眼中闪过一丝"、"心中涌起一股"、"嘴角勾起一抹"、"不禁XXX" | 出现1个即失败 |
| **古龙式极简** | 词字独立成段（"痒。""疼。"） | 出现1个即失败 |
| **时间连接词** | "然后"、"就在这时"、"过了一会儿" 开头 | 出现≥3个即失败 |
| **抽象统计词** | "无数"、"成千上万" | 出现≥2个即失败 |
| **精确年龄** | "XX岁的XXX" | 出现≥2个即失败 |
| **Markdown加粗** | `**XXX**` | 出现1个即失败 |

**输出格式**：
```
【禁止项检测】
AI味表达：X个 [示例]
古龙式极简：X个 [示例]
时间连接词：X个 [示例]
抽象统计词：X个 [示例]
精确年龄：X个 [示例]
Markdown加粗：X个 [示例]
结果：通过/失败
```

## 三级权重体系

> **新增**：借鉴 InkOS 的三级权重体系，实现更灵活的问题判定

### 权重级别定义

| 级别 | 英文 | 影响 | 处理方式 |
|------|------|------|----------|
| 🔴 **Critical** | 关键 | `passed = false` | 自动进入修订循环，必须修复 |
| 🟡 **Warning** | 警告 | `passed = true` | 记录问题，可人工审核后决定是否修订 |
| 🟢 **Info** | 信息 | 不影响判定 | 仅记录，作为改进建议 |

### 判定规则

```
passed = true  ⟺  无 Critical 级别问题
passed = false ⟺  存在任意 Critical 级别问题
```

### 维度七：信息边界检查 🔴 Critical

> **来源**：InkOS 维度 9（Information Boundary）

| 检查内容 | 标准 | 权重 |
|---------|------|------|
| **信息越界** | 角色不能知道其未接触的信息 | 🔴 Critical |
| **信息传递链** | 信息从 A→B 有明确传递过程 | 🔴 Critical |
| **秘密保持** | 角色的秘密在被揭示前不应被其他角色知晓 | 🔴 Critical |
| **预言/伏笔信息** | 角色不能提前知道伏笔揭示后的信息 | 🟡 Warning |

**检测规则**：
1. 检查角色是否"记起"从未亲眼见过的事
2. 检查角色是否引用了其他角色才知道的信息
3. 检查角色是否对未知信息做出准确判断

**输出格式**：
```
【信息边界检查】
信息越界：__处 [✓/🔴]
信息传递链：__条 [✓/🔴]
秘密保持：__项 [✓/🔴]

问题详情（如有）：
- 🔴 角色A知道了角色B的秘密，但无传递过程 → 需补充信息传递场景
- 🔴 角色C"记起"从未见过的事件 → 需修改为合理记忆来源
```

### 维度八：大纲偏离检测 🔴 Critical

> **来源**：InkOS 维度 33（Outline Drift）

| 检查内容 | 标准 | 权重 |
|---------|------|------|
| **剧情节点匹配** | 与 `章节大纲/*.md` 规划一致 | 🔴 Critical |
| **节奏跨度** | 规划 N 章内容不能 1-2 章讲完 | 🔴 Critical |
| **核心事件覆盖** | 本章核心事件是否完成 | 🟡 Warning |
| **支线推进** | 支线是否按计划推进 | 🟡 Warning |

**检测规则**：
1. 对照章节大纲，检查核心情节点是否覆盖
2. 检查是否有情节被跳过或过度压缩
3. 检查是否有新增未规划的重要情节

**输出格式**：
```
【大纲偏离检测】

新增情节：__个 [需评估/正常]

问题详情（如有）：
- 🔴 核心情节"__"被跳过 → 需补充或调整大纲
- 🔴 规划3章内容1章讲完 → 节奏过快，建议扩展
- 🟡 新增情节"__"未在大纲中 → 需评估是否合理
```

### 维度十三：场景拼接逻辑检查 🔴 Critical

| **物体状态连续性** | 物体状态变化必须合理（绳子上吊后数量等） | 🔴 Critical |
| **人物行为逻辑** | 角色行为必须符合情境（敌未走远不出现等） | 🔴 Critical |
| **场景元素一致性** | 同一场景的元素在不同段落必须一致 | 🔴 Critical |

#### 详细检查清单

##### 1. 人物数量一致性检查

```yaml
检查规则:
  - 妇孺数量: 场景A出现数量 == 场景B出现数量（除非有死亡/离开事件）
  - 断后人数: 断后场景人数 == 尸体场景人数
  - 佣兵数量: 追踪人数 == 肢解人数

常见错误:
  - 场景A: "七个妇人，五个孩子" → 场景B: "三个女的，两个孩" ❌
  - 场景A: "十几个男人断后" → 场景B: "五具尸体" ❌
```

##### 2. 时间线逻辑检查

```yaml
检查规则:
  - 等待逻辑: 角色出藏身处前，威胁必须已经离开
  - 因果顺序: 结果必须在原因之后
  - 时间跨度: 明确的时间等待必须合理

常见错误:
  - "佣兵的脚步声渐渐远去" → 立刻"爬出树洞" ✓
  - "佣兵还在附近" → "爬出树洞" ❌
  - "趴了整整一个时辰" → 合理等待 ✓
  - "等了三秒就出来" → 不合理 ❌
```

##### 3. 场景转换逻辑检查

```yaml
检查规则:
  - 必须包含环节: 场景A → [过渡] → 场景B
  - 空间连续性: 角色移动路径合理
  - 视角转换: POV角色变化时需要过渡

场景顺序示例:
  正确: 出树洞 → 母亲残骸 → 铁牙尸体 → 村口 → 男人残肢
  错误: 出树洞 → 直接到村口（跳过母亲残骸和铁牙） ❌
```

##### 4. 性别代词一致性检查

```yaml
检查规则:
  - 同一角色全程使用相同性别代词
  - 血牙 = 男性 = "他"
  - 母亲 = 女性 = "她"

常见错误:
  - 场景A: "血牙站在树洞里" 
  - 场景B: "她站起来" ❌（血牙变成女性）
```

##### 5. 物体状态连续性检查

```yaml
检查规则:
  - 绳子数量: 上吊人数 == 绳子数量
  - 尸体残骸: 肢解后必须有残骸描写
  - 物品变化: 物品状态变化必须合理

常见错误:
  - "十几个身影上吊" → "只有一根绳子" ❌
  - "肢解完成" → "没有尸体残骸" ❌
  - "剥皮完成" → "地上干干净净" ❌
```

##### 6. 人物行为逻辑检查

```yaml
检查规则:
  - 出现时机: 威胁未消除时不应出现
  - 行为动机: 角色行为必须有合理动机
  - 情感反应: 角色必须有合理的情感反应

常见错误:
  - 佣兵还在附近 → 血牙就爬出来 ❌
  - 目睹母亲被肢解 → 没有情感描写 ❌
  - 看到父亲尸体 → 直接走过不看 ❌
```

##### 7. 攻击类型一致性检查

```yaml
检查规则:
  - 佣兵势力构成: 不同势力使用对应攻击类型
  - 攻击类型匹配: 修仙者→飞剑，魔法师→火球，科技战士→脉冲枪

常见错误:
  - 大纲: "飞剑、脉冲、火球三路攻击"
  - 正文: "只有骑兵冲锋" ❌
```

### 评估前技法检索

在开始评估前，**必须**按优先级获取评估标准：

```python
import sys
from pathlib import Path

sys.path.insert(0, r"D:\动画\众生界\.vectorstore")

# 技法文件路径（回退用）
TECHNIQUE_DIR = Path(r"D:\动画\众生界\创作技法")

# 维度到文件映射（回退用）
DIMENSION_FILE_MAP = {
    "世界观": "01-世界观维度/02-世界观维度.md",
    "剧情": "02-剧情维度/03-剧情维度.md",
    "人物": "03-人物维度/04-人物维度.md",
    "战斗": "04-战斗冲突维度/05-战斗冲突维度.md",
    "氛围": "05-氛围意境维度/06-氛围意境维度.md",
    "叙事": "06-叙事维度/07-叙事维度.md",
    "主题": "07-主题维度/08-主题维度.md",
    "情感": "08-情感维度/09-情感维度.md",
    "读者体验": "09-读者体验维度/10-读者体验维度.md",
}

def get_evaluation_standards(scene_type: str, evaluation_dimensions: List[str]):
    """获取评估标准 - 按优先级：数据库优先，文件回退"""
    
    from workflow import NovelWorkflow
    workflow = NovelWorkflow()
    
    dimension_map = {
        "历史纵深": "世界观",
        "内在逻辑一致性": "世界观",
        "世界观自生长": "世界观",
        "命运驱动": "剧情",
        "历史回响": "剧情",
        "伏笔跨度": "剧情",
        "群像塑造": "人物",
        "独立命运线": "人物",
        "道德灰色": "人物",
        "选择代价": "人物",
        "有代价胜利": "战斗",
        "群体牺牲有姓名": "战斗",
        "战术逻辑": "战斗",
        "历史沉淀感": "氛围",
        "静默叙述": "氛围",
        "语言重量感": "氛围",
    }
    
    standards = []
    
    for dim in evaluation_dimensions:
        dimension = dimension_map.get(dim, "")
        
        # ===== Step 1: 数据库检索（优先） =====
        try:
            results = workflow.search_techniques(
                query=f"{scene_type} {dim} 标准",
                dimension=dimension,
                top_k=2
            )
            
            if results and len(results) > 0:
                # 数据库检索成功，使用结果
                for result in results:
                    standards.append({
                        "评估维度": dim,
                        "技法名称": result.get("名称", "未知"),
                        "技法内容": result.get("内容", ""),
                        "来源": result.get("维度", "") + "维度",
                        "检索方式": "数据库"
                    })
                continue  # 成功，跳过文件回退
        except Exception as e:
            print(f"[数据库检索失败] {dim}: {e}")
        
        # ===== Step 2: 文件回退（数据库失败时） =====
        if dimension in DIMENSION_FILE_MAP:
            file_path = TECHNIQUE_DIR / DIMENSION_FILE_MAP[dimension]
            if file_path.exists():
                try:
                    with open(file_path, "r", encoding="utf-8") as f:
                        content = f.read()
                    standards.append({
                        "评估维度": dim,
                        "技法名称": f"{dimension}维度技法",
                        "技法内容": content[:500],  # 截取前500字
                        "来源": str(file_path),
                        "检索方式": "文件回退"
                    })
                    print(f"[文件回退] {dim}: {file_path.name}")
                except Exception as e:
                    print(f"[文件读取失败] {file_path}: {e}")
    
    return standards
```

### 写入内容

```python
import json
from pathlib import Path
from datetime import datetime

def write_chapter_log(chapter_name: str, evaluation_result: dict):
    """
    将评估结果写入章节经验日志
    
    Args:
        chapter_name: 章节名称，如 "第一章-天裂"
        evaluation_result: Evaluator 输出的评估结果
    """
    log_dir = Path("D:/动画/众生界/章节经验日志")
    log_dir.mkdir(parents=True, exist_ok=True)
    
    # 提取章节号
    chapter_num = extract_chapter_number(chapter_name)
    log_file = log_dir / f"第{chapter_num}章_log.json"
    
    # 构建日志内容
    log_content = {
        "chapter": chapter_name,
        "created_at": datetime.now().isoformat(),
        
        # 使用的技法
        "techniques_used": extract_techniques_used(evaluation_result),
        
        # 有效做法
        "what_worked": evaluation_result.get("反馈", {})
            .get("洞察提取", {})
            .get("有效做法", []),
        
        # 无效做法
        "what_didnt_work": evaluation_result.get("反馈", {})
            .get("洞察提取", {})
            .get("无效做法", []),
        
        # 可复用洞察
        "insights": evaluation_result.get("反馈", {})
            .get("洞察提取", {})
            .get("可复用洞察", []),
        
        # 给下一章的建议
        "for_next_chapter": evaluation_result.get("反馈", {})
            .get("洞察提取", {})
            .get("给下一章建议", [])
    }
    
    # 写入文件
    with open(log_file, "w", encoding="utf-8") as f:
        json.dump(log_content, f, ensure_ascii=False, indent=2)
    
    print(f"[经验日志] 已写入: {log_file}")
    return log_file

def extract_techniques_used(evaluation_result: dict) -> list:
    """从评估结果中提取使用的技法"""
    techniques = []
    technique_scores = evaluation_result.get("技法评估", {})
    
    for dimension, scores in technique_scores.items():
        if isinstance(scores, dict):
            for tech_name, score_info in scores.items():
                effect = "有效" if score_info.get("评分", 0) >= 7 else "待优化"
                techniques.append({
                    "name": tech_name,
                    "effect": effect,
                    "note": score_info.get("说明", "")
                })
    
    return techniques
```

## 关键原则（更新）

1. **分离原则** - 与 Generator 完全分离
2. **具体反馈** - 提供可操作的修改建议
3. **技法引用** - 评估结果必须引用具体技法
4. **硬性阈值** - 禁止项有明确的通过/失败标准
5. **调优要求** - 根据实际效果调整阈值和评估标准
6. **洞察沉淀** - 从评估中提取可复用的经验，写入章节日志
